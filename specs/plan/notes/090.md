# WI-090: Fix run_extraction config param and merge _write_beats_and_log

## Part 1: Fix config parameter

In `run_extraction()` at line 68, change:
```python
cfg: dict = resolve_config(cwd)  # type: ignore[assignment]
```
to:
```python
cfg: dict = config if config is not None else resolve_config(cwd)  # type: ignore[assignment]
```

This eliminates the double config load when cb_extract passes config.

## Part 2: Merge _write_beats_and_log

Add optional `beats` parameter to run_extraction():
```python
def run_extraction(transcript_text, session_id, trigger, cwd, config=None, beats=None):
```

When `beats` is provided:
- Skip transcript parsing and LLM extraction (the `extract_beats()` call)
- Use the provided beats directly
- Continue with the same vault writing, autofile, dedup, run log, journal logic

When `beats` is None (default):
- Current behavior: parse transcript, call LLM, get beats

Then delete `_write_beats_and_log()` entirely (lines ~398-484).

Update `main()` --beats-json path (~line 289) to call:
```python
result = run_extraction(
    transcript_text="",  # not used when beats provided
    session_id=args.session_id,
    trigger=args.trigger,
    cwd=args.cwd,
    config=config,
    beats=beats,
)
```

## Key detail
The `transcript_text` parameter is still required positionally. When beats are provided, it can be empty string since it won't be used. Alternatively, make it optional too — but that changes the signature for all callers. The empty string approach is simpler.
