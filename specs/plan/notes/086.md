# WI-086: Migrate relation vocabulary to merged 7-predicate set

## Changes

### vault.py (VALID_PREDICATES, ~line 164)
Replace:
```python
VALID_PREDICATES = {"related", "references", "broader", "narrower", "supersedes", "wasderivedfrom"}
```
With:
```python
VALID_PREDICATES = {"related", "references", "causes", "caused-by", "supersedes", "implements", "contradicts"}
```

Note: `wasDerivedFrom` was already lowercased to `wasderivedfrom` in VALID_PREDICATES (fixed in WI-029). The new set uses all-lowercase with hyphens for multi-word predicates.

### extract-beats-system.md (~lines 88-104)
Update the relation predicate documentation to list the new vocabulary with definitions:
- `related` — general association between notes
- `references` — this note cites or refers to the target
- `causes` — this note describes something that led to the target
- `caused-by` — this note was caused by the target
- `supersedes` — this note replaces the target (target is outdated)
- `implements` — this note realizes a decision or plan from the target
- `contradicts` — this note conflicts with the target

### resolve_relations() in vault.py
The function already normalizes unknown predicates to "related". Verify this behavior is preserved with the new set. No logic change needed — just the set contents change.

### Tests
Update any tests that assert specific SKOS predicates (broader, narrower, wasDerivedFrom) to use new vocabulary predicates.
