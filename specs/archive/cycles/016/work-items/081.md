# WI-081: Clarify README MCP JSON for both install paths

## Problem

The MCP JSON example at ~line 441 was updated in cycle 015 to use `uv run --directory` pattern. This serves the plugin install path but doesn't clarify what Claude Desktop manual-install users should use.

## Fix

Add two labeled JSON blocks:

1. **Plugin installation** (recommended): `uv run --directory ${CLAUDE_PLUGIN_ROOT}` pattern
2. **Manual installation** (install.sh fallback): the path that install.sh actually writes

For manual install, `install.sh` copies files to `~/.claude/cyberbrain/` and the MCP server is launched via `mcp/start.sh` which finds `uv` and runs the server. The correct manual config is:
```json
{
  "mcpServers": {
    "cyberbrain": {
      "command": "/path/to/cyberbrain/mcp/start.sh"
    }
  }
}
```

Or the direct uv invocation:
```json
{
  "mcpServers": {
    "cyberbrain": {
      "command": "uv",
      "args": ["run", "--directory", "/path/to/cyberbrain", "python", "-m", "cyberbrain.mcp.server"]
    }
  }
}
```

Check what `install.sh` actually writes to Claude Desktop config and use that as the manual example.
