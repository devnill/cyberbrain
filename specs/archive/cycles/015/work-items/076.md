# WI-076: Fix autofile.py bare import

## Problem

`autofile.py` line 469 uses `from search_index import update_search_index` — a bare module name that fails with `ImportError` in the packaged src-layout. The `except Exception` on line 473 silently swallows this, so autofile-created notes are never indexed via this code path.

## Fix

Change line 469 from:
```python
from search_index import update_search_index  # noqa: I001  # type: ignore[import-not-found]  # optional runtime dependency
```
to:
```python
from cyberbrain.extractors.search_index import update_search_index
```

Remove the `# type: ignore[import-not-found]` annotation since the import is no longer broken. Keep the `# noqa: I001` if needed for import ordering. The `except Exception` block should remain but update the comment to remove the `ImportError if not installed` note — the import is no longer expected to fail for that reason.

## Testing

Add a test that verifies `update_search_index` is called after a successful autofile create action. The existing test infrastructure mocks `call_model` and provides a temp vault — mock `update_search_index` and assert it's called with the correct path.
