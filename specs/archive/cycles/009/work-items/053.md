# WI-053 Implementation Notes: Standalone vault frontmatter repair script

## Purpose

One-time (or repeatable) repair tool for vault notes that already have accumulated duplicate
frontmatter fields from the WI-052 defect. The WI-052 fix prevents future duplicates; this
script repairs existing ones.

## Location

`scripts/repair_frontmatter.py` — standalone, no MCP server required.

## Config Loading

Read vault path from `~/.claude/cyberbrain/config.json` → `vault_path` key.
Fall back to `--vault PATH` CLI flag if provided or if config is missing.

```python
import json, os, argparse
from pathlib import Path

CONFIG_PATH = Path.home() / ".claude" / "cyberbrain" / "config.json"

def get_vault_path(args):
    if args.vault:
        return Path(args.vault)
    config = json.loads(CONFIG_PATH.read_text())
    return Path(config["vault_path"])
```

## Duplicate Detection

A frontmatter block has duplicate keys if any key appears more than once in the lines between
opening `---` and closing `---`. Detection algorithm:

1. Read file text
2. Check `text.strip().startswith("---")` and `text.find("\n---", 3) != -1`
3. Extract frontmatter lines
4. Collect keys: `line.split(":", 1)[0].strip()` for lines containing `:`
5. If `len(keys) != len(set(keys))` → duplicates present

## Deduplication

Keep the **last occurrence** of each key (consistent with pyyaml `yaml.safe_load()` behavior —
what callers currently see). This means the most recently enriched values survive, which is
the intent.

Algorithm:
1. Walk frontmatter lines in order, tracking `{key: line}` dict (later lines overwrite earlier)
2. Reconstruct frontmatter preserving only the surviving line for each key, in the order of
   their **last occurrence**
3. Prepend `---\n`, append `\n---`
4. Concatenate with body (everything after closing `---`)

## CLI Interface

```
python scripts/repair_frontmatter.py [--apply] [--vault PATH]

Options:
  --apply     Write repaired frontmatter to disk (default: dry-run, no writes)
  --vault     Override vault path (default: read from ~/.claude/cyberbrain/config.json)
```

## Output Format

```

  [DRY RUN] Would repair: Projects/cyberbrain/Some other note.md
    - type (2 duplicates → 1)

Summary: 847 notes scanned, 23 with duplicate fields, 0 repaired (dry-run)
Run with --apply to write repairs.
```

When `--apply`:
```
Summary: 847 notes scanned, 23 with duplicate fields, 23 repaired
```

## Safety

- Default mode is dry-run — no accidental writes
- Non-frontmatter content (body) is never touched
- Script does not call any LLM or MCP tool
- If a file can't be read (permissions, encoding), log a warning and continue
- If the repaired frontmatter would be identical to the original (no change), skip silently
