**scripts/test.py:**
```python
#!/usr/bin/env python3
"""Smart test runner: quiet first pass, detailed on failure."""

import subprocess
import sys
from pathlib import Path


def main():
    full_suite = "--full" in sys.argv

    # Pass 1: Quick check
    cmd = ["pytest", "--tb=no", "-q", "--no-header"]
    if not full_suite:
        cmd.append("--affected-only")

    result = subprocess.run(cmd, capture_output=True, text=True)

    if result.returncode == 0:
        # Extract pass count from last line
        summary = result.stdout.strip().split("\n")[-1] if result.stdout.strip() else "All tests passed"
        print(f"✓ {summary}")
        return 0

    # Pass 2: Detailed on failure (only failed tests)
    summary = result.stdout.strip().split("\n")[-1] if result.stdout.strip() else "Tests failed"
    print(f"✗ {summary}")
    print("\nRe-running failed tests with detail...")

    subprocess.run(["pytest", "--last-failed", "--tb=short", "-v"])
    return 1


if __name__ == "__main__":
    sys.exit(main())
```

**Make executable:**
```bash
chmod +x scripts/test.py
```

**Usage:**
```bash
# Targeted tests (only affected by changes), quiet
python scripts/test.py

# Full test suite, quiet
python scripts/test.py --full

# Direct pytest access (bypass wrapper)
pytest --tb=short -v  # Normal verbose
pytest --tb=long -v    # Full detail
```

The wrapper should:
1. Default to `--affected-only` mode
2. Run quiet first pass
3. Only show detail on failure
4. Use pytest's `--last-failed` to re-run only failures
