# Vault Guide

This vault stores knowledge captured from development sessions, organized by project and topic area.

## Folder Structure

- **AI/Claude-Sessions/** — General-scope captured notes
- **AI/Working Memory/** — Ephemeral notes for current project state
- **AI/Journal/** — Daily journals
- **Projects/** — Project-specific notes organized by project name

## Beat Types

| Type | Description |
|---|---|
| `decision` | A choice made between alternatives, with rationale |
| `insight` | A non-obvious understanding or pattern discovered |
| `problem` | Something broken, blocked, or constrained |
| `reference` | A fact, command, snippet, or configuration detail |

## Conventions

- Use descriptive titles that stand alone without context
- Tags should be lowercase, hyphenated, specific
- Include relations when notes connect to existing knowledge
- Working memory notes are for current project state, not long-term knowledge

## Cyberbrain Preferences

- Prefer concise summaries over verbose descriptions
- Always include code snippets when discussing implementation details
- Tag architecture decisions with the system component they affect
- Working memory notes should capture the current state of in-progress work, not general knowledge
