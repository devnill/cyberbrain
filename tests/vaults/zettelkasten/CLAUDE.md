# Vault Guide

This vault follows Zettelkasten principles: atomic notes, extensive linking, and emergent structure.

## Conventions

- Each note captures exactly one idea
- Filenames are timestamp-prefixed: `YYYYMMDDHHMMSS Title.md`
- Notes link heavily via `related` frontmatter and inline wikilinks
- No deep folder hierarchy — flat or lightly categorized
- Structure emerges from links, not folders
- Hub notes aggregate related atomic notes

## Beat Types

| Type | Description |
|---|---|
| `decision` | A choice made between alternatives, with rationale |
| `insight` | A non-obvious understanding or pattern discovered |
| `problem` | Something broken, blocked, or constrained |
| `reference` | A fact, command, snippet, or configuration detail |

## Tagging

- Tags are descriptive, not hierarchical
- Prefer specific tags over broad categories
- Use tags to create cross-cutting views across the link graph
