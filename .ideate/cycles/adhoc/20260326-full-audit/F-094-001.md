**Reviewer**: Claude Sonnet 4.6  
**Date**: 2026-03-26

## Verdict: Fail

Two significant findings: `update_vault_note` silently creates files rather than updating existing ones (semantic contract violation), and `shared.py` now has a duplicate `_is_within_vault` implementation that diverges from vault.py's version.

## Significant Findings

### S1: `update_vault_note` does not verify the target file exists

- **File**: `src/cyberbrain/extractors/vault.py:457-462`
- **Issue**: `update_vault_note` calls `path.write_text()` without first checking `path.exists()`. The function's docstring says "Overwrite an existing vault note" — but it silently creates a new file if the note does not exist. This is a semantic contract violation: callers that believe they are updating a known note may instead be creating a phantom note at a typo'd or stale path, with no error raised.
- **Impact**: Silent data loss scenario — the caller's intended target note is not updated; a new file is created at the wrong path instead. No test catches this because the test suite only covers the happy path where the file pre-exists.
- **Suggested fix**: Add an existence check before writing:
  ```python
  def update_vault_note(path: Path, content: str, vault_path: str) -> Path:
      vault = Path(vault_path)
      _is_within_vault_check(vault, path)
      if not path.exists():
          raise FileNotFoundError(f"Note does not exist: {path}")
      path.write_text(content, encoding="utf-8")
      return path
  ```
  Add a corresponding test: `test_raises_if_note_does_not_exist`.

### S2: Duplicate `_is_within_vault` in `shared.py` is now a divergence risk

- **File**: `src/cyberbrain/mcp/shared.py:95-101`
- **Issue**: `shared.py` retains its own `_is_within_vault(vault, target) -> bool` implementation (lines 95-101) that is byte-for-byte identical to the one in `vault.py` (lines 98-104). The new `_is_within_vault_check` in `vault.py` is a third variant (raises instead of returns). All three implement the same `resolve().relative_to()` logic. Now that `vault.py` owns the canonical validation, `shared.py`'s copy should be deleted and replaced with an import — otherwise a future behaviour change (e.g. symlink handling) will need to be applied in two places independently.
- **Impact**: Maintenance divergence. The callers in `restructure/execute.py` and `restructure/pipeline.py` import `_is_within_vault` from `shared`, while the new write functions use `_is_within_vault_check` from `vault.py`. The two silently differ in return type (bool vs raise), which makes it non-obvious which guard to use in new code.
- **Suggested fix**: Remove `_is_within_vault` from `shared.py` and import it from `vault.py`:
  ```python
  from cyberbrain.extractors.vault import (
      _is_within_vault,
      _is_within_vault_check,
      move_vault_note,
      update_vault_note,
      write_beat,
      write_vault_note,
  )
  ```
  The existing callers in restructure modules import from `shared`, so the re-export keeps their import paths unchanged.

## Minor Findings

### M1: `move_vault_note` does not guard against overwriting an existing destination

- **File**: `src/cyberbrain/extractors/vault.py:465-472`
- **Issue**: `src.rename(dest)` on POSIX silently replaces `dest` if it already exists (POSIX `rename(2)` semantics). The function has no guard. The existing `write_beat` function explicitly checks for collisions with a counter loop; `move_vault_note` does not.
- **Suggested fix**: Add a pre-condition check or document the overwrite behaviour explicitly:
  ```python
  if dest.exists():
      raise FileExistsError(f"Destination already exists: {dest}")
  ```
  If silent overwrite is intentional, add a docstring note. Either way, add a test for the collision case.

### M2: No test for `update_vault_note` when the file does not exist

- **File**: `tests/test_vault.py:50-72`
- **Issue**: The `TestUpdateVaultNote` class has no test for the missing-file case. Because `update_vault_note` currently doesn't raise on a missing file (see S1), this gap directly masks the bug.
- **Suggested fix**: Once S1 is fixed, add:
  ```python
  def test_raises_if_note_does_not_exist(self, tmp_path):
      vault = tmp_path / "vault"
      vault.mkdir()
      missing = vault / "ghost.md"
      with pytest.raises(FileNotFoundError):
          update_vault_note(missing, "content", str(vault))
  ```

### M3: No test for `move_vault_note` when `src` does not exist

- **File**: `tests/test_vault.py:75-123`
- **Issue**: There is no test verifying that `move_vault_note` raises when `src` does not exist. On POSIX, `Path.rename()` raises `FileNotFoundError` natively, but this is an undocumented behaviour assumption. An explicit test documents the contract.
- **Suggested fix**: Add:
  ```python
  def test_src_not_found_raises(self, tmp_path):
      vault = tmp_path / "vault"
      vault.mkdir()
      with pytest.raises(FileNotFoundError):
          move_vault_note(vault / "ghost.md", vault / "dest.md", str(vault))
  ```
